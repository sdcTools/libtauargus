Technische documentatie NewTauArg.Dll

NewTauArgus is een COM DLL die functies en events bevat voor het beveiligen van tabel-bestanden. Als ontwikkelomgeving is gebruikt Visual C++ 6.0.

Voor het beveiligen zijn de volgende instrumenten beschikbaar:
  • Detecteren van onveilige primaire tabelcellen
    Dominantieregel
    Gegeven een percentage p en een aantal a: een cel is onveilig als de som van de a grootste bijdragers aan de cel groter of gelijk is aan p% van het totaal van de cel. Voor de bijdragen aan de cel wordt de zogenaamde shadow-variabele gebruikt.
    PQ-regel
    Stel de bijdragen aan een cel zijn geordend van groot naar klein x1, x2, x3, ..., N het aantal bijdragen in de cel en T het celtotaal. Eis: een respondent mag een andere respondent tot op p% nauwkeurig kunnen schatten.
    De 'worst case' is nummer 2 mag nummer 1 slechts tot op p% kunnen schatten. Zijn beste schatting voor x1 is T - x2. De eis is nu dat (T - x2) - x1 ≥ (p / 100) * x1

    Hierin is p ≤ q en n ≥ 1.
  • Detecteren van onveilige secundaire tabelcellen
    GHMITER
    Een programma ontwikkeld door het Landesamt für Datenverarbeitung und Statistik. Een zeer uitgebreide beschrijving ervan is te vinden in het document GHMITE22i PC.doc.
    HITAS
    Een programma ontwikkeld door PP en JJ. Zie A heuristic approach to cell-suppression in hierarchical tables
    van Peter-Paul de Wolf, BPA-nummer 1569-99-RSM.
Voor het beveiligen zijn de volgende events en functies beschikbaar.
Elke functie en event bevat:
  1. Een korte beschrijving
  2. De parameters zijn van type:
    • VARIANT = variant datatype (meestal gebruikt om een string van arrays door tegeven als input)
    • BSTR = string (teruggave en input)
    • double = 8-bytes real
    • long = 4-bytes integer
    • VARIANT_BOOL = VARIANT_TRUE of VARIANT_FALSE (alleen gebruikt als return value)
    • short = 2-bytes integer
  3. Return value

SetVariable

Beschrijving:
Specificeert de eigenschappen van een variabele.
Parameters:
long Index 1, 2, ... n_var, index van variabele
long bPos 1, 2, ... startpositie in elk record van het invoerbestand
long nPos 1, 2, ... aantal posities in elk record van het invoerbestand, max. 100
long nDec 0,1,.... aantal decimalen (vooral van belang bij schrijven veilig bestand)
LPCTSTR  Missing1 code voor eerste missing (alleen van belang bij categorale variabele)
LPCTSTR Missing2 code voor tweede missing (alleen van belang bij categorale variabele)
BOOL IsCategorical  is een categoriale variabele
BOOL IsNumeric  is een numerieke variabele
BOOL IsWeight  is een gewicht (moet ook eigenschap IsNumeric hebben)
BOOL IsHierarchical  is een hiërarchische variabele
BOOL IsHolding  is een variabele die een holding identificeert; alle opeenvolgende records met een gelijke code worden geacht tot eenzelfde holding te behoren
Voor categorale variabelen: Als Missing1 leeg is wordt Missing2 gepromoveerd. Als beide leeg zijn wordt false teruggegeven. Als beide gelijk zijn wordt Missing2 geacht leeg te zijn.
Voor niet-categorale variabelen zijn Missings irrelevant.
Slechts één variabele mag de eigenschap IsHolding dan wel IsWeight hebben.
Een variabele met eigenschap IsWeight mag niet (IsCategorical of IsHierarchical) zijn.
Een variabele met eigenschap IsHolding mag niet (IsCategorical of IsNumeric of IsWeight of IsHierarchical) zijn.
Return value:
false als een of meer parameters fout zijn, anders true

top

SetHierarchicalDigits

Beschrijving:
Specificeert de eigenschappen van de codes van een hiërarchische variabele, moet plaatsvinden vóór de aanroep van ExploreFile.
Kan niet samengaan met SetHierarchicalCodelist. De som van de array nDigits moet gelijk zijn aan nPos (zie SetVariable). Het aantal groepjes mag niet meer dan 10 zijn.
De codelijst van een hiërarchische variabele wordt vastgesteld uit het invoerbestand én uit de gegeven hiërarchie.
Bijvoorbeeld voor een code van 5 tekens met verdeling {2,1,2}
  • 12
  • 123
  • 12301
  • 12302
  • 12340
  • 12345
  • 12349
  • 124
  • 12401
  • 12402
  • 12403
  • 12404
  • 12405
Alle 5-cijferig codes stonden in het invoerbestand, de andere zijn daaruit afgeleid volgens de verdeling {2,1,2}. Code 12 is de som van code 123 en 124; code 123 is de som van code 12301 t/m 12349, code 124 is de som van code 12401 t/m 12405.
Parameters:
long Index 1, 2, ... n_var, index van variabele
long nDigitPairs   aantal groepjes met elk een zeker aantal tekens
Array long nDigits Array van nDigitsPairs elementen met elk het aantal tekens van de hiërarchie.
Return value:
false als een of meer parameters fout zijn, anders true

DoRecode

Beschrijving:
Vermindert het aantal codes van een variabele door meerdere codes samen te nemen. In de RecodeString staan, gescheiden door newlinetekens (\r\n), regels met achtereenvolgens doelcode, ':' en broncodes.
Broncodes die te smal zijn worden verbreed door er spaties voor te zetten.
Codes mogen omgeven worden door aanhalingstekens ("3- "), dit kan nodig zijn als een code eindigt op een spatie of als een code een komma of een streepje bevat.

Een code wordt altijd als een alfanumerieke behandeld. Daarom is bijvoorbeeld de code "1a" groter dan "19" en de code "1 " kleiner dan "11".

Bijvoorbeeld:
  0 : - 90 alle codes kleiner dan of gelijk aan 90
  1 : 91 - 500 codes vanaf 91 tot en met 500
  2 : 501 - codes groter dan of gelijk aan 501
  3 : 11, "3 ", 512, 530-570, 930-970  meerdere series toegestaan, scheidingsteken ','
Parameters:
long  VarIndex  1, 2, ... n_var, index van variabele
LPCTSTR  RecodeString De specificatie van de hercodering
LPCTSTR Missing1 De waarde van Missing1
LPCTSTR Missing2 De waarde van Missing2
⇒long  ErrorType Type fout
⇒long  ErrorLine Regel waarin de fout is geconstateerd
⇒long  ErrorPos Positie waarop de fout is geconstateerd
⇒BSTR  WarningString  Waarschuwing voor overlappende broncodes, niet genoemde codes enz.
Return value:
false als een of meer parameters fout zijn (bijv. geen categoriale variabele, er blijft niets van een code over), anders true

De labels worden veiligheidshalve omsloten door dubbele aanhalingstekens, ze kunnen immers een komma bevatten. Staan er dubbele aanhalingstekens in dan worden die vervangen door enkele.
Parameters:
long  TableIndex  Index tabel (1, 2, ...)
LPCTSTR  FileName  Naam van het resultaatbestand
Array long  DimSequence  Volgorde waarin dimensies worden behandeld; eerste .. in de laag, voorlaatste in de rij, laatste in de kolom. Dus bij een driedimensionale tabel met DimSequence = [3,2,1] komt Tabeldimensievariabele 3 in de laag-, idem 2 in de rij-, idem 1 in de kolomdimensie
Return value:
false als er iets fout ging, anders true

top

WriteCellRecords

Beschrijving:
Schrijft voor elke cel van de tabel een record met achtereenvolgens de dimensiecodes en de bijbehorende cel, gescheiden door een komma. De dimensiecodes worden omsloten door dubbele aanhalingstekens, staan er dubbele aanhalingstekens in dan worden die vervangen door enkele.
Parameters:
long  TableIndex  Index tabel (1, 2, ...)
LPCTSTR  FileName  Naam van het resultaatbestand
BOOL  SuppressEmpty  Lege cellen niet (true) of wel (false) schrijven
Return value:
false als er iets fout ging, anders true

SetHierarchicalCodelist

Beschrijving:
Specificeert de eigenschappen van de codes van een hiërarchische variabele, moet plaatsvinden vóór de aanroep van ExploreFile.
Kan niet samengaan met SetHierarchicalDigits.

LPCTSTR  FileName  Naam van het bestand met de hiërarchische codelijst; de niveaus worden aangegeven door het aantal LevelStrings aan het begin van een regel. Een generaal-totaal wordt geacht niet aanwezig te zijn, dat wordt door het programma zelf toegevoegd als eerste (lege) code van een codelijst. Bijvoorbeeld:
